# Release Notes

Use this skill for changelogs, release notes, version summaries, and PR summaries.

## Inputs

Use available sources in this order:

1. User-provided summary or scope
2. Merged PRs
3. Commit history
4. Diff summary

## Structure

Group notes as relevant:

- Features
- Fixes
- Breaking changes
- Internal changes
- Known issues

## Writing Guidance

- Keep language clear and customer-readable.
- Avoid implementation detail unless the audience is technical.
- Call out breaking changes explicitly.
- Do not overstate unverified impact.

## Output Summary

Provide a concise release note draft and any source gaps that could affect accuracy.
